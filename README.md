# Expense Tracker API

Сервис для отслеживания личных финансов на языке Go. Позволяет регистрировать доходы и расходы, просматривать историю транзакций и получать ежемесячную статистику.

## Особенности
- **Архитектура**: Clean Architecture (Domain, Service, Repository, Handler).
- **База данных**: PostgreSQL.
- **Миграции**: Автоматическое применение миграций при запуске через Docker.
- **Контейнеризация**: Поддержка Docker и Docker Compose.
- **Конфигурация**: Управление через переменные окружения (`.env`).

## Технологии
- Go 1.22+
- PostgreSQL 18
- Docker / Docker Compose
- Makefile
- Библиотеки: `pgx` (драйвер БД), `uuid`, `sqlmock` (для тестов).

## Настройка окружения

Перед запуском необходимо создать файл `.env` с настройками. В корне проекта подготовлен шаблон `.env.example`.

```bash
# Скопируйте шаблон в реальный .env файл
cp .env.example .env
```

Затем откройте `.env` и отредактируйте параметры (порт, доступы к БД), если это необходимо.

## Как запустить

### 1. Через Docker Compose
Это самый простой способ, который поднимет базу данных, применит миграции и запустит API.
```bash
docker-compose up --build
```
API будет доступно по адресу `http://localhost:8080` (или по порту, указанному в `.env`).

### 2. Локальный запуск
Для этого способа у вас должен быть установлен Go и запущен PostgreSQL.

```bash
# Установка зависимостей
go mod download

# Запуск приложения через Makefile (автоматически подтянет .env)
make run
```

## Управление проектом через Makefile
В проекте настроен `Makefile` для автоматизации рутинных задач:

- `make run` — запуск приложения.
- `make build` — компиляция бинарного файла.
- `make test` — запуск всех тестов.
- `make test-cover` — запуск тестов с генерацией HTML-отчета о покрытии.
- `make migrate-up` — применить миграции к БД.
- `make migrate-down` — откатить последнюю миграцию.

## Эндпоинты API

### Создание транзакции
**POST** `/api/transactions`
```json
{
  "type": "expense",
  "amount": 1500,
  "category": "Food",
  "description": "Lunch at cafe",
  "date": "2024-03-20T15:04:05Z"
}
```

Поля `description` и `date` указывать необязательно. В случае, если не указан `description`, поле с описанием будет пустым. В случае, если не указан `date`, будет браться текущая дата и время.

### Получение списка транзакций
**GET** `/api/transactions`

### Получение статистики
**GET** `/api/statistics?year=2024&month=3`
Параметры `year` и `month` необязательны (по умолчанию берется текущий период).

## Тестирование
Проект включает unit-тесты для всех слоев. Чтобы проверить покрытие кода:
```bash
make test-cover
```
После выполнения откройте созданный файл `coverage.html` в браузере.
